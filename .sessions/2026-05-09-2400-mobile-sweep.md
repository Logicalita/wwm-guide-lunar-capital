## User request

Scan the codebase and add adjustments for the mobile version. Several components have shipped over recent sessions without their own mobile rules.

## Plan / decisions

- **Scope**: targeted CSS-only fixes for the components added since the original 768px media query was written. No HTML, JS, or new components — just tightening layouts that get cramped on phones.
- **Two-tier breakpoints**: kept the existing `@media (max-width: 768px)` for tablet/sidebar transitions; added a new `@media (max-width: 480px)` below it for narrow-phone fixes only. Keeps tablets unaffected.
- **Pain points fixed** (in the new 480px block):
  - **Case-stats 3-column → 1-column stack**. The four cards in "Tại sao min thuộc tính khác hệ = best" each have a `case-stats` 3-tile grid (Min/Max/Trung bình). Numbers like "1054.3" got squeezed at phone widths. Stacking to one column trades vertical space for legibility; the highlighted "Trung bình" tile keeps its color treatment because `.highlight` still applies.
  - **Damage-example bars: stack label above bar+value**. The `dmg-example .dmg-r` desktop layout uses a 130px label column to fit "Sát thương yếu", which leaves the absolute-probability bar with too little width on phones. Switched to a CSS grid with `grid-template-areas` so the label sits on its own row above the bar+value pair. Bar gets full width; value (50px) anchors to the right.
  - **Damage-flow SVG scroll affordance**. The SVG already scrolls horizontally on phones via `overflow-x: auto + min-width: 580px`, but there's no visual cue. Added a `::after` pseudo-element on `.dmg-flow-wrap` that prints "← lướt ngang →" centered below the SVG. Pure CSS, no HTML change.
  - **Tighter card padding**. `.card`, `.case-card`, `.dmg-example`, and `.dmg-example-head` lose a few px of padding to claw back inner content width on 320px phones.
  - **Smaller headings**. `h1` drops from 28px (768px) to 24px (480px); `.key-num` drops from 52 to 44; hero padding tightens from `36px 18px 30px` to `28px 14px 24px`. Long Vietnamese titles like `Hướng dẫn build chí mạng Tank` no longer push hero height excessively.
- **Deliberately NOT touched**: the recommend-page `.dmg-r` rows (short labels work fine), `.summary-bars`, `.toc-item`, `.warn-card`, `.compare`, `.pri` rows, `.case-head`, sidebar dropdown indent, and SVG text-size scaling — these all read OK on phones with the existing rules. Keeping scope tight; if real-device testing surfaces issues, follow up.
- **Why 480px (and not 600px or 414px)**: 480px is the conventional "phone but not the smallest" breakpoint and matches where `repeat(3, 1fr)` tiles start to feel cramped at typical 8–10 char numbers in a serif font.

## Files changed

- `styles.css` — appended a new `@media (max-width: 480px) { … }` block immediately after the existing `@media (max-width: 768px)` block. Block contains: `.case-stats` stacking rule; `.dmg-example .dmg-r` grid-area restack with three child grid-area assignments; `.dmg-flow-wrap::after` scroll-hint pseudo-element; tighter `.card` / `.case-card` / `.dmg-example` / `.dmg-example-head` padding; smaller `h1` / `.key-num` / `.hero` for narrow widths.

## Open follow-ups

Couldn't do that from this session — the user should tap-test the dropdown nav, scroll the SVG horizontally, and skim the case-cards on an actual phone. If anything still looks cramped, file a follow-up.
- The SVG `::after` hint always shows under the wrapper at ≤480px even when the SVG happens to fit (rare since min-width: 580 > 480). Acceptable since the hint is small; could be more clever with container queries later if it bothers anyone.
- The damage-example bars use absolute width (a 36% bar is literally 36% of the track). On the new stacked mobile layout the track gets wider, which makes small probabilities (e.g. 10% Sát thương yếu) more readable. Win-win.
